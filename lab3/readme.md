## Сервисы


| Сервис              | Порт | Назначение                                     |
| ------------------- | ---- | ---------------------------------------------- |
| **postgres**        | 5432 | База данных|
| **user-service**    | 8081 | Регистрация, логин, JWT, поиск пользователей   |
| **hotel-service**   | 8082 | Список отелей, поиск по городу, создание отеля |
| **booking-service** | 8083 | Создание брони, мои брони, отмена брони        |

## Контракты

Находятся в openapi.yaml

## Аутентификация

- Выдача токена: `POST /v1/auth/login` (user-service).
- Проверка через middleware **Bearer JWT** (`libs/auth`), защищены операции с отелями (создание) и всё API бронирований.

## SQL

| Файл | Назначение |
|------|------------|
| **`schema.sql`** | Таблицы `users`, `hotels`, `bookings`, констрейнты, индексы |
| **`data.sql`** | Тестовые данные |
| **`queries.sql`** | Примеры SQL|
| **`optimization.md`** | Оптимизация запросов|

## Запуск

Через докер:

```bash
cd lab2
docker compose build
docker compose up -d
```

Тесты:

```bash
chmod +x tests/smoke.sh
./tests/smoke.sh
```

Только перед повторным тестированием надо удалить образ постгреса, потому что есть тест на создание пользака, он может упасть, так как есть проверки на уникальность
