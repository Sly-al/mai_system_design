# Оптимизация запросов

## 1. Поиск пользователя по логину

Запрос: `SELECT ... FROM users WHERE login = $1`.

Seq sacn всей таблицы `users` при отсутствии индекса на `login`. Добавил ограничение `UNIQUE`, под капотом создаётся b-tree индекс, планировщик использует index scan, запрос отрабатывает быстрее.

## 2. Список отелей по городу

Запрос: `SELECT ... FROM hotels WHERE city = $1 ORDER BY id`.

Добавил индекс `idx_hotels_city` по полю `city`, запрос также стал отрабатывать быстрее

## 3. Бронирования пользователя

Запрос: `SELECT ... FROM bookings WHERE user_id = $1 ORDER BY id`.

Также как и выше добавил индекс `idx_bookings_user_id`. Это позволяет взять строки без seq scan-а на всю таблицу.

## 4. Проверка пересечения активных броней по отелю

Запрос из `queries.sql`: `EXISTS (...)` по `hotel_id`, `status = 'active'` и условию на даты.

Индекс `idx_bookings_hotel_active_dates` является частичным и тем самым уменьшает объём читаемых строк по сравнению с seq scan-ом всех броней отеля.
